# GitHub 上传前重要提示

## ⚠️ 图片文件夹重命名

**在上传到 GitHub 之前，必须将图片文件夹重命名！**

### 原因
GitHub Pages 对中文文件夹路径的支持不够稳定，可能导致图片无法显示。

### 操作步骤

1. **重命名文件夹**
   - 将文件夹 `图` 重命名为 `images`
   - 在 Windows 中：右键点击文件夹 → 重命名 → 输入 `images`

2. **验证路径**
   - 所有 HTML 文件中的图片路径已经更新为 `images/`
   - 例如：`<img src="images/龙岗区地图.png">`

3. **上传到 GitHub**
   - 确保上传的是重命名后的 `images` 文件夹
   - 不是原来的 `图` 文件夹

## 📁 正确的文件结构

上传到 GitHub 后，文件结构应该是：

```
.
├── index.html
├── styles.css
├── script.js
├── README.md
├── .gitignore
└── images/              ← 必须是这个名称！
    ├── 龙岗区地图.png
    ├── 龙岗高程.png
    ├── 坡度.png
    └── ... (其他图片)
```

## ✅ 检查清单

上传前确认：
- [ ] 文件夹已从 `图` 重命名为 `images`
- [ ] HTML 文件中的路径都是 `images/`（不是 `图/`）
- [ ] 所有图片文件都在 `images` 文件夹内
- [ ] 图片文件名保持中文（可以，但文件夹名必须是英文）

## 🔍 如果图片还是不显示

1. **检查 GitHub 仓库中的文件结构**
   - 进入仓库页面
   - 确认有 `images` 文件夹（不是 `图`）
   - 点击进入 `images` 文件夹，确认所有图片都在

2. **检查图片路径**
   - 在浏览器中打开网站
   - 按 F12 打开开发者工具
   - 查看 Console（控制台）是否有 404 错误
   - 查看 Network（网络）标签，看哪些图片加载失败

3. **常见问题**
   - 如果看到 404 错误，检查路径是否正确
   - 如果路径中有 `图/`，说明 HTML 文件没有更新
   - 如果文件夹名还是 `图`，需要重命名

4. **清除缓存**
   - GitHub Pages 更新后，等待 1-2 分钟
   - 清除浏览器缓存后刷新

## 📝 快速修复命令（如果使用 Git）

如果已经上传了错误的文件夹名，可以使用以下命令：

```bash
# 重命名文件夹
git mv 图 images

# 提交更改
git commit -m "重命名图片文件夹为images以兼容GitHub Pages"

# 推送到GitHub
git push
```
